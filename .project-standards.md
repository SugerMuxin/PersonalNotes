# 项目标准规范

## 文件创建标准

从2026年1月14日起，本项目中的**所有新创建文件**必须遵守以下标准：

### 1. Properties（YAML Frontmatter）

每个Markdown文件必须包含以下属性：

```yaml
---
author: RuoSaChen
tags:
  - [相关标签1]
  - [相关标签2]
---
```

### 2. 封面图片

每个文件必须在Properties后添加封面图片引用：

```markdown
![[cover-xxx.jpg/png]]
```

可用封面图片分类：
- `cover-economy.jpg` - 经济类
- `cover-history.png` - 历史类
- `cover-unity.jpg` - 技术类
- `cover-life.png` - 生活类
- `cover-read.webp` - 阅读类
- `cover-personal.jpg` - 个人类
- `cover-per.png` - 个人计划类
- `cover-ai.jpg` - AI相关
- `cover-shader.png` - 图形渲染
- `cover-en.jpg/png` - 英文/综合

### 3. 文件组织规范

- 放置在合适的子目录下（如Ecnomic、Daily、Articles等）
- 使用清晰的文件命名
- 保持目录结构整洁
